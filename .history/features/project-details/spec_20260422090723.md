## Work
- Describe the UI/UX clearly and specifically: The website is a single column scrolling (mobile first). The content is laid top to bottom.
- What elements exist? What container elements exist?: Header is the same as the home page with navigation links, a back link to go back to the home page. Then we'll have the body, with a project row, the title, process maybe, view code button, images of the project and such. Then we'll have the footer which will be generally the same as the home page, but we'll have a next project button that'll load the next project. (Might change that behavior later on)
- How do users interact? What is the hover behavior?: The users interact with the back link to go back to the home page, external links to view the code, gallery images can be expanded (this will be done for polish), next project link (not 100% sure yet), and the nav links.
  
## Deliverables
- What does “done” look like?: Done for me would have to mean the layout is completely functional, and it'd look nice to look at the website and has reached my minimalistic and clean goal.
